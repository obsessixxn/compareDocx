# compareDocx

Скрипт для сравнения документов из ТЗ с документами из ЛК Клиента. Находит расхождения в текстовом содержимом и формирует отчёт.

## Поддерживаемые форматы

- `.docx`
- `.doc` 
- `.pdf`

## Структура проекта

```
compareDocx/
├── compare_docs.py   # основной скрипт
├── requirements.txt  # зависимости
├── folder_tz/        # документы из ТЗ (эталон)
├── folder_lk/        # документы из ЛК Клиента
├── debug/            # дамп нормализованного текста (при DEBUG=True)
└── report.txt        # отчёт о расхождениях (генерируется автоматически)
```

## Установка

```bash
pip install -r requirements.txt
```

## Использование

1. Поместите документы из ТЗ в папку `folder_tz/`.
2. Поместите документы из ЛК Клиента в папку `folder_lk/`.
3. Файлы в обеих папках должны иметь **одинаковые имена** (без учёта расширения)
4. Запустите скрипт:

```bash
python compare_docs.py
```

## Результат

- Отчёт выводится в консоль.
- Если есть расхождения — дополнительно сохраняется в `report.txt`.
- При `DEBUG = True` (по умолчанию) нормализованный текст каждого документа сохраняется в папку `debug/` для ручной проверки.

## Статусы в отчёте

| Статус | Значение |
|--------|----------|
| `✓ Совпадает с эталоном` | Текст ЛК полностью совпадает с ТЗ |
| `✗ ОТЛИЧАЕТСЯ от эталона` | Найдены расхождения (diff прилагается) |
| `✗ ОТСУТСТВУЕТ в ЛК` | Документ есть в ТЗ, но отсутствует в ЛК |
| `! Лишний файл в ЛК` | Документ есть в ЛК, но отсутствует в ТЗ |
| `? Ошибка чтения` | Не удалось прочитать файл |


## Минусы

1. Не понимает сравнение таблиц, нужно проверить вручную
2. Не может сравнивать параграфы в .pdf и в .docx, поэтому в отличия может упасть что-то типа:
"2"
"2.1"
Из-за разницы в отступах
